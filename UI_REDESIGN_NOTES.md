# 口袋記帳 UI 改版需求

最後更新：2026-04-19

## 改版目標

這個 App 在手機上的第一優先要是：

`一打開就可以立刻記一筆`

目前首頁把登入與雲端同步資訊放得太前面，會打斷記帳流程。下一版要改成更像「個人記帳工具」，而不是先看到設定面板。

## 首頁優先順序

首頁從上到下建議調整為：

1. `快速記一筆`
2. `本月概況`
3. `最近記錄`
4. `其他功能入口`

其中：

- `快速記一筆` 要放在最前面
- 使用者打開 App 時，不需要先滑過登入資訊
- 表單應該是首頁最容易碰到的區塊

## 雲端登入區塊

目前「登入與資料位置」區塊太大，資訊量太多。

下一版請改成：

- 縮成較小的狀態卡
- 只保留必要資訊
- 不要搶首頁主要視覺

建議顯示內容：

- 目前模式：`本機模式` / `雲端模式`
- 帳號：登入後只顯示 email 簡短資訊
- 一個主按鈕：
  - 未登入：`Google 登入`
  - 已登入：`管理同步`

如果需要顯示詳細說明，可以：

- 收進展開區塊
- 或放到次頁 / 設定頁

## 手機版首頁體驗

手機版首頁要優先服務「快速輸入」。

預期體驗：

- 打開就能看到輸入表單
- 只要選支出 / 收入、輸入金額、分類、日期，就能送出
- 不要先被大段同步說明擋住

## 每月花費資訊

目前每月統計可以保留，但詳細花費不要佔據首頁太多空間。

下一版建議：

- 首頁只顯示精簡摘要
- 詳細月花費改成：
  - 一個按鈕點開
  - 或獨立面板
  - 或切換頁面後查看

## 視覺化建議

每月花費顯示可以改得更直覺，不一定要全部用文字卡片。

可選方向：

1. `圓餅圖`
   - 顯示各分類支出占比
   - 適合快速看本月錢花去哪

2. `像 token 用量圖的比例圖`
   - 以條狀 / 環狀方式顯示分類占比
   - 重點是「一眼看出最大宗花費」

3. `首頁顯示摘要，詳細資料另外展開`
   - 首頁保留：
     - 本月支出
     - 本月收入
     - 結餘
   - 詳細分類圖在按鈕後再開

## 建議首頁資訊層級

### 第一層：立即操作

- 快速記一筆

### 第二層：快速理解現況

- 本月收入
- 本月支出
- 本月結餘

### 第三層：延伸查看

- 本月分類占比圖
- 詳細月花費
- 雲端同步狀態
- 備份 / 匯出 / 匯入

## 具體調整方向

### A. 把快速記一筆移到最上方

這是最高優先。

### B. 把登入同步區縮小

不要再用大型說明卡佔首頁第一屏。

### C. 詳細月花費移出首頁核心區

首頁只留最重要的月摘要。

### D. 加入圖像化統計

優先考慮：

- 圓餅圖
- 分類比例圖

## 下一版實作建議

可以依照這個順序做：

1. 重新排列首頁區塊順序
2. 縮小同步卡
3. 把本月詳細花費改成次要入口
4. 加入分類占比圖
5. 微調手機版間距與按鈕大小

## 一句話總結

下一版首頁要從「設定導向」改成「記帳導向」。
